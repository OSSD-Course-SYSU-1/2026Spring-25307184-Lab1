# CommonEventAndNotification 工程文件描述

## 1. 工程概述

**CommonEventAndNotification** 是一个基于 **HarmonyOS** 平台的闹钟应用示例工程，主要演示鸿蒙系统中 **公共事件（CommonEvent）** 与 **通知（Notification）** 功能的使用。该工程通过系统级提醒代理（`reminderAgentManager`）和通知管理（`notificationManager`）实现闹钟的创建、管理、触发和通知推送。

- **Bundle名称**: `com.huawei.alarmclock`
- **应用版本**: 1.0.0（versionCode: 1000000）
- **编译SDK**: HarmonyOS 6.0.2(22)
- **运行环境**: HarmonyOS
- **开发语言**: ArkTS（鸿蒙 TypeScript 扩展）
- **架构模式**: MVVM（Model-View-ViewModel）
- **版权**: Huawei Device Co., Ltd（Apache License 2.0）

## 5. 核心业务代码分析

### 5.1 入口 Ability（EntryAbility.ets）

应用启动入口，继承自 `UIAbility`：

- **`onCreate()`**: 初始化全局变量，包括 `bundleName`、`abilityName`、`abilityWant`，并创建 `PreferencesHandler` 单例用于数据持久化。
- **`onWindowStageCreate()`**: 获取屏幕显示信息，初始化偏好设置数据库，加载 `pages/MainIndex` 作为默认页面。

### 5.2 公共事件与通知服务（ReminderService.ets）★核心类

这是整个工程中与"CommonEvent和Notification"主题最核心的类：

#### 功能一：通知权限管理
- **`openNotificationPermission()`**: 请求用户开启通知权限（`requestEnableNotification`），并添加 `SERVICE_INFORMATION` 类型的通知槽位（`notification.addSlot`），用于后续闹钟提醒的通知通道。

#### 功能二：闹钟提醒发布（公共事件）
- **`addReminder(alarmItem, callback)`**: 
  - 构建 `ReminderRequestAlarm` 提醒请求对象
  - 调用 `reminderAgent.publishReminder()` 向系统发布提醒（即发布公共事件）
  - 返回系统分配的 `reminderId`
  - 支持的配置项：提醒类型、小时/分钟、标题、响铃时长、再响次数、再响间隔、操作按钮（关闭/稍后提醒）、通知ID等

#### 功能三：闹钟提醒取消
- **`deleteReminder(reminderId)`**: 调用 `reminderAgent.cancelReminder()` 取消系统提醒

#### 提醒对象核心配置：
```typescript
reminderAgent.ReminderRequestAlarm {
  reminderType: item.remindType,     // 提醒类型（闹钟）
  hour: item.hour,                    // 小时
  minute: item.minute,                // 分钟
  title: item.name,                   // 提醒标题
  ringDuration: ...,                  // 响铃时长（秒）
  snoozeTimes: ...,                   // 再响次数
  timeInterval: ...,                  // 再响间隔（分钟）
  actionButton: [                     // 提醒操作按钮
    { title: '关闭', type: ACTION_BUTTON_TYPE_CLOSE },
    { title: '稍后提醒', type: ACTION_BUTTON_TYPE_SNOOZE }
  ],
  wantAgent: {                        // 点击跳转目标
    pkgName: globalThis.bundleName,
    abilityName: globalThis.abilityName
  },
  notificationId: ...,                // 关联通知ID
  slotType: notification.SlotType.SERVICE_INFORMATION  // 通知槽位类型
}
```

### 5.3 数据持久化（PreferencesHandler.ets）

基于鸿蒙 `@kit.ArkData` 的 `preferences` 轻量级键值对数据库：

| 方法 | 功能 |
|------|------|
| `getInstance()` | 单例模式获取实例 |
| `configure(context)` | 初始化 preferences 并注册数据变更监听 |
| `set(key, value)` | 异步存储数据（含 flush 持久化） |
| `get(key, defValue)` | 异步读取数据 |
| `addPreferencesListener(listener)` | 添加数据变更监听器（观察者模式） |

- **存储Key**: `alarmData`（存储所有闹钟数据的JSON字符串）

## 7. 数据流与架构

```
用户操作
   │
   ▼
┌──────────┐     ┌──────────────┐     ┌─────────────────────┐
│  View    │ ──► │  ViewModel   │ ──► │  ReminderService    │
│ (.ets UI)│ ◄── │ (.ets逻辑)   │     │  (发布/取消提醒)     │
└──────────┘     └──────┬───────┘     └──────────┬──────────┘
                        │                        │
                        │                 ┌──────▼──────────┐
                        ▼                 │ 鸿蒙系统          │
               ┌──────────────┐          │ - reminderAgent  │
               │Preferences   │          │ - notification   │
               │Handler       │          │   Manager        │
               │(数据持久化)   │          └─────────────────┘
               └──────────────┘
```

1. **View层**（`MainIndex.ets`/`DetailIndex.ets`及子组件）响应用户交互
2. **ViewModel层**（`MainViewModel.ets`/`DetailViewModel.ets`）处理业务逻辑
3. **Model层**：
   - `ReminderService.ets`: 调用系统级提醒代理和通知管理API（**CommonEvent + Notification 核心实现**）
   - `PreferencesHandler.ets`: 通过 preferences 实现数据持久化
4. 数据变更通过观察者模式（`PreferencesListener`）通知UI更新

## 8. 功能特性总结

| 特性 | 说明 |
|------|------|
| **闹钟创建/编辑** | 选择时间、设置重复日、响铃日期、时长、间隔、名称 |
| **闹钟启用/禁用** | 通过Toggle开关控制，底层发布/取消系统提醒 |
| **闹钟删除** | 从数据中移除并取消系统提醒 |
| **系统提醒触发** | 利用 `reminderAgent.publishReminder` 发布公共事件 |
| **通知推送** | 创建通知槽位（`SERVICE_INFORMATION`），绑定通知ID |
| **操作按钮** | 提醒弹出时提供"关闭"和"稍后提醒"操作 |
| **数据持久化** | 通过 Preferences 存储闹钟数据，应用重启不丢失 |
| **模拟时钟** | Canvas渲染的指针式时钟，点击可切换为数字显示 |
| **屏幕适配** | `DimensionUtil` 基于设计稿360×780比例适配 |
